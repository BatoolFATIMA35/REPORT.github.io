1.1         Abstract

DBpedia is a community that extract structure data.This is the community effort to extract structured information from Wikipedia. Wikipedia do not clearly define datatype of attributes. So, their problems were solved by using DBpedia mapping based extraction approach. So DBpedia ontology was developed and all attributes were mapped to the properties but the problem is that ontology classes, data types and properties are not mapped in Urdu on the other hand are mapped in most languages like English, Japanese, German and many more. In this project we provide the recipe of Urdu DBpedia. It will be a contribution in DBpedia international community. It will facilitate those people whose native language is Urdu. As a new chapter it will be added to the multilingual DBpedia chapter, which can support the maintenance, extraction and enriching of DBpedia. It has the ability to answer enrich query. This system will also go through the maximum available resources and provide exact data and definition. We will also do mapping of all classes, datatypes and properties in Urdu. It will also help apple and Facebook because It has converted into a central knowledge base within Link Open Data in future it will also help in their voice assistance service for user of Urdu.

1.3         Project Methodology

 

Every language uses different techniques to develop DBpedia. For example, Korean DBpedia used same techniques and extraction frameworks that have already been used by English Dbpedia. Urdu DBpedia does not have too much data at this time and it will be built by using Different steps:

1-     Mapping of all ontology classes, datatypes and properties in Urdu language. It will be done manually because currently there is no software which can help Urdu dbpedia Mapping

2-     It will be checked either Info boxes exist in article or no. If Info boxes are not available then they will be created first. If some attribute values are missing then fill them first. Mapping of the Urdu Wikipedia Info boxes to the DBpedia ontology.

3-     Creation of new ontology classes and visualization to work on different projects.

This will help them to get more accurate google search result in Urdu also the companies like Facebook and Nadra can use Urdu dbpedia database to get more information regarding Urdu things.

2.  Overall System Description

The overall system will be developed through manual work by mapping ontologies classes, properties and data types in Urdu languages. This is time consuming project but to achieve high goals this is justified. This project is basically developed on Wikipedia data so we can say that it will be dependent on Wikipedia. The major constraint is that you have to do all work manually no framework is available right now for Urdu dbpedia.

2.1  User characteristics

This system is developed for all users who search on google but we specially work on Urdu dbpedia so it will help only Urdu native speakers. This system will help them by showing more accurate google results.

2.2  Operating environment

The operating requirement for this project and for user is to have WIFI connection and google search engine for searching. This system environment will be enough for general public to have accurate search results.

2.3  System constraints

The major constraint of dbpedia:

a) Many classes do not have comments. Many people complain about it.

b) Its difficult to investigate events like many words are map event.

c)No validation of data.

d)Framework does not work for many languages.

e) No specific properties like area have property but we do not know which area land area or something else.

3.  External Interface Requirements

This project Urdu dbpedia has almost no external components because we need to do mapping of ontology that is already pre-defined in dbpedia we just need to do Urdu mapping. There is no direct connection between this project and user. We will do our work on backend user will get accurate result. There are some areas where we need to use open link data where we will use sparql or other languages.

3.1  Hardware Interfaces

This system will support all devices with internet support and google search engine. There is not specific hardware requirement for user. There is not hardware interfaces for this project.

3.2  Software Interfaces

We will use protg to create new classes and subclasses for Urdu native users. We will use languages to query all data bases like sparql. In this project many data bases can share data. We will get our data from Wikipedia and many other resources. Mostly the data that we exchange between the software or applications will be the definitions of data. We can work on NLG in future after completing mapping of ontologies.

For the application programming interface availability DBpedia is the most reliable and its structure data is even used by famous companies like google and Microsoft. Its SPARQL language is available at every time. The user will get accurate results due to dbpedia because it will provide structure data that will extracted from Wikipedia and will also give results on base of ontology relationships.

2-Safety Requirement:

Overall system is secure it cannot harm user directly. If you want to contribute to dbpedia you have to take permission for mapping and for new classes creation. The most possible harm user can have been he will get wrong search result. Ambiguity and inconsistency among many issues can arise. The solution for this mostly people argue for the clear explanation of the standards which can be created with semantic technologies such as ontologies. However, this possibility has gained less attention.

3-Security requirement:

There is no such security requirement for user but for contributor or developer you have to take authentication from dbpedia. This way the only authorize person can change or work on dbpedia.
